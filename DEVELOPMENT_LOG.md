# บันทึกประวัติการพัฒนาและแก้ไขระบบ (Development & Debugging Log)

เอกสารฉบับนี้จัดทำขึ้นโดย **Antigravity** เพื่อบันทึกประวัติการวิเคราะห์ แก้ไขปัญหาระบบแสดงผล และการติดตั้งฟีเจอร์วัดประสิทธิภาพระบบปฏิบัติการเวลาจริง (RTOS) บนโปรเจกต์แม่แบบ STM32F407xxx

---

## 1. ปัญหาที่พบและการแก้ไข (Bug Fixes)

### ปัญหาที่ 1: ตัวอักษรตัวพิมพ์เล็กแสดงผลบิดเบี้ยว (ASCII Character Shift Bug)
* **อาการ:** ตัวพิมพ์ใหญ่ ตัวเลข และสัญลักษณ์สเปซแสดงผลปกติ แต่**ตัวพิมพ์เล็กทั้งหมดจะแสดงผลเลื่อนไป 1 ตัวอักษรเสมอ** เช่น ข้อความในโค้ดคือ `"DSP Core: Active"` แต่บนจอ OLED แสดงผลเป็น `"DSP Cpsf: Adujwf"` (ตัวสะกดเลื่อนตามลำดับ ASCII เช่น `o` -> `p`, `e` -> `f`)
* **สาเหตุ:** พบปัญหาความขัดแย้งพรีโปรเซสเซอร์ (C Preprocessor Gotcha) ในไฟล์ `Inc/font.h` บรรทัดที่ 75:
  ```c
  0x02, 0x04, 0x08, 0x10, 0x20, // \
  ```
  เครื่องหมาย backslash `\` ที่เขียนไว้ในคอมเมนต์บรรทัดสุดท้าย ถูกคอมไพเลอร์ GCC ตีความว่าเป็น **ตัวเชื่อมบรรทัด (Line Continuation)** ทำให้บรรทัดถัดไป (บรรทัดที่ 76 ซึ่งเป็นข้อมูลของตัวอักษรปิดก้ามปู `]`) ถูกควบรวมและถูกลบออกไปเนื่องจากถือว่าเป็นส่วนหนึ่งของคอมเมนต์บรรทัดก่อนหน้า
  การสูญเสียข้อมูลตัวอักษร `]` ทำให้ขนาดของอาร์เรย์ `Font6x8_Data` ขาดหายไป 5 ไบต์ และดึงข้อมูลตัวพิมพ์เล็กตัวถัดๆ ไปมาแสดงแทนที่
* **การแก้ไข:** แก้ไขคอมเมนต์ในไฟล์ `Inc/font.h` จาก `// \` เป็น `// backslash` เพื่อตัดสถานะการเชื่อมบรรทัด และขยายขนาดข้อมูลอาร์เรย์กลับมาครบถ้วนที่ **475 ไบต์ (95 ตัวอักษร)** ส่งผลให้ตัวพิมพ์เล็กทั้งหมดแสดงผลถูกต้อง

### ปัญหาที่ 2: จุดพิกเซลกะพริบที่ขอบซ้ายสุดของจอเมื่อวัตถุชนขอบขวาสุด (OLED Boundary Wrap-around Bug)
* **อาการ:** เมื่อลูกบอลเด้งเคลื่อนที่ไปสัมผัสขอบหน้าจอฝั่งขวา (พิกัด $x \approx 124$) จะเกิดจุดพิกเซลเล็กๆ กะพริบขึ้นที่ขอบจอฝั่งซ้ายสุด (คอลัมน์ 0 และ 1)
* **สาเหตุ:** ในฟังก์ชันส่งหน้าข้อมูลผ่าน DMA (`SSD1306_StartPageTransfer` ในไฟล์ `Src/ssd1306.c`) มีการตั้งค่าที่อยู่คอลัมน์เริ่มต้นส่งไว้ที่คอลัมน์ที่ 2 (`0x02`) เพื่อจุดประสงค์สำหรับชิป SH1106
  ทว่าหน้าจอจริงของผู้ใช้เป็นระบบ **SSD1306** (มี 128 คอลัมน์พอดี) เมื่อส่งข้อมูล 128 ไบต์เริ่มจากคอลัมน์ที่ 2 ส่งผลให้ไบต์ที่ 126 และ 127 ล้นขอบเขตแสดงผลและ**วนกลับไปเขียนลงที่ตำแหน่งคอลัมน์ 0 และ 1 ของหน้าจอ**
* **การแก้ไข:** แก้ไขที่อยู่คอลัมน์เริ่มต้นในไฟล์ `Src/ssd1306.c` จาก `0x02` เป็น `0x00` เพื่อให้จัดหน้าข้อมูล 128 คอลัมน์ตรงกับพิกเซลจริงของจอแบบ 1-to-1 ส่งผลให้ภาพไม่บิดเบี้ยว และไม่มีจุดวนย้อนกลับไปกะพริบที่ขอบซ้ายสุดอีก

## 2. ฟีเจอร์ที่เพิ่มเติม (New Features)

เพื่อสนับสนุนงานทดสอบเชิงประยุกต์ด้านประมวลผลสัญญาณ (DSP) และระบบปฏิบัติการเวลาจริง (RTOS) จึงมีการติดตั้งระบบวิเคราะห์ประสิทธิภาพภายใน (RTOS Telemetry) แสดงผลทางหน้าจอฝั่งซ้าย:

1. **ระบบวัดอัตราการทำงานของหน่วยประมวลผล (CPU Usage):**
   * เปิดใช้ฟีเจอร์รวบรวมสถิติรันไทม์ใน `FreeRTOSConfig.h`
   * ตั้งค่าไทเมอร์ฮาร์ดแวร์ **TIM5** (32-bit Timer) ให้ทำงานเป็นตัวเก็บสถิติประมวลผลกลาง ทำงานความถี่ 20 kHz (ละเอียดทุกๆ 50 ไมโครวินาที)
   * คำนวณเวลาประมวลผลจริงเทียบกับเวลาทำงานของ **IDLE Task** ทุกๆ 1 วินาที เพื่อแสดงผลเป็นเปอร์เซ็นต์ภาระงาน
2. **มาตรวัดแรมฮีปและจำนวนงาน (Free Heap & Active Tasks Telemetry):**
   * ดึงจำนวนแรมฮีปที่เหลือว่างของระบบผ่านคำสั่ง `xPortGetFreeHeapSize()`
   * ดึงจำนวน Task ที่จดทะเบียนรันอยู่ผ่านคำสั่ง `uxTaskGetNumberOfTasks()` (ซึ่งแสดงผล **5 Tasks** ประกอบด้วย: `IDLE`, `Tmr Svc`, `OLED_UI`, `DSP_Test`, และ `Load_Test`)
3. **งานจำลองโหลดเพื่อการทวนสอบ (Sync Load Test Task):**
   * เขียนฟังก์ชัน `load_test_task` ที่จะสลับการประมวลผลทุกๆ 3 วินาที โดยจะทำงานหนัก (Busy Loop) กินโหลด CPU 100% พร้อมจองหน่วยความจำ Heap ขนาด 20 KB เป็นเวลา 1 วินาทีเต็ม เพื่อให้ผู้ใช้สังเกตสภาวะการขยับขึ้นลงประสานกันของ CPU และ Heap บนหน้าจอ OLED

## 3. สรุปการปรับปรุงเชิงสถาปัตยกรรมทุกบรานช์ (Architecture Alignment)

การแก้ไขและอัปเกรดฟีเจอร์ด้านบนทั้งหมด รวมถึงการ**ปรับความเร็วสัญญาณนาฬิกาบัส I2C1 เป็น 400 kHz (Fast Mode)** ได้รับการตรวจสอบ คอมไพล์ผ่านอย่างสมบูรณ์ และพุช (Push) ขึ้นสู่ GitHub ในลักษณะเดียวกันครบถ้วนทั้ง 3 บรานช์เป้าหมาย ได้แก่:

* `feature/oled-with-dma-double-buffer` (ระบบสลับหน้าจอคู่ Double Buffer ไม่เกิด Screen Tearing)
* `feature/oled-with-dma-single-buffer` (ระบบบัฟเฟอร์เดี่ยวประหยัดแรม ปิดลูปประมวลผลด้วย Task Notification)
* `feature/oled-without-dma` (ระบบส่งค่าแบบธรรมดาผ่าน CPU Polling)
